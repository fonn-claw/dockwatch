**1. Create `src/lib/actions/work-orders.ts`:**
- "use server" directive
- Import from "zod/v4" (NOT "zod")

**createWorkOrder(formData: FormData):**
- requireRole(["manager"])
- Zod schema: title (string 1-255), description (optional string), type (enum: preventive/corrective/inspection/emergency), priority (enum: urgent/high/normal/low), dockId (number), assetId (optional number), assigneeId (optional number), dueDate (optional string), notes (optional string)
- Insert into workOrders with createdById=session.userId, status defaults to "created"
- If assigneeId is provided, also set status to "assigned" (auto-assign)
- Call logAudit: action="create", entityType="work_order", entityId=newId, metadata={ title, type, priority }
- revalidatePath("/work-orders")
- Return { success: true, id }

**transitionWorkOrder(id: number, newStatus: string, notes?: string):**
- requireRole(["manager", "crew"])
- Fetch current work order by id
- Call canTransition(wo.status, newStatus) — if false, return { error: `Cannot transition from ${wo.status} to ${newStatus}` }
- If newStatus === "verified" and session.role !== "manager", return { error: "Only managers can verify work orders" }
- Build updates object: { status: newStatus, updatedAt: new Date() }
- If newStatus === "in_progress": set startedAt = new Date()
- If newStatus === "completed": set completedAt = new Date()
- If newStatus === "verified": set verifiedAt = new Date(), verifiedById = session.userId
- **Backward transitions:** If going backward (not isForwardTransition), clear downstream timestamps:
  - completed->in_progress: set completedAt = null
  - in_progress->assigned: set startedAt = null
  - assigned->created: set assigneeId = null (per research recommendation)
- Update work order
- Call logAudit: action="transition", entityType="work_order", entityId=id, metadata={ from: wo.status, to: newStatus, notes }
- revalidatePath("/work-orders") and revalidatePath(`/work-orders/${id}`)
- Return { success: true }

**2. Create `src/lib/queries/work-orders.ts`:**

**getWorkOrders(filters: { dockId?: number; status?: string; priority?: string; assigneeId?: number; dueDateFrom?: string; dueDateTo?: string; assetId?: number }, userRole: string, userId: number):**
- Build conditions array starting with [isNull(workOrders.deletedAt)]
- If userRole === "crew", add eq(workOrders.assigneeId, userId) — crew sees only assigned orders
- Apply all optional filters: dockId, status, priority, assigneeId (for manager filtering), dueDateFrom (gte), dueDateTo (lte), assetId
- Query with relations: `with: { dock: true, asset: true, assignee: true, createdBy: true }`
- Order by: priority (urgent first) then dueDate asc (soonest first)
- Return results

**getWorkOrderDetail(id: number):**
- Query single work order with all relations: dock, asset, assignee, createdBy, verifiedBy, parts
- Return result or null

**getCrewUsers():**
- Query users where role is "crew" or "manager" (potential assignees), ordered by name
- Return array of { id, name, role }
