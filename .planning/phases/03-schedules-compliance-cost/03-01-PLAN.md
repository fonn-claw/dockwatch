- Export `ASSET_TYPE_TO_CATEGORY` mapping: piling->structural, electrical_pedestal->electrical, water_connection->plumbing, dock_light->electrical, fire_extinguisher->safety, fuel_pump->mechanical, cleat->structural, bumper->structural, gangway->structural, other->general

**Create `src/lib/queries/schedules.ts`:**
- `getSchedules(filters)` — Query maintenanceSchedules with optional filters for assetType, frequency, season, isSafetyCritical, complianceStatus (on_track/due_soon/overdue). Join asset for asset name and dock name. Return array with computed fields: compliancePercent (count of on-time completed WOs / total due WOs for this schedule), statusColor (green if nextDueAt > 7 days out, yellow if within 7 days, red if past due).
- `getScheduleStats()` — Return totals: activeCount, overdueCount, dueSoonCount, onTrackCount, avgCompliancePercent. Overdue = nextDueAt < now. DueSoon = nextDueAt within 7 days. OnTrack = nextDueAt > 7 days.
- Use Drizzle ORM query patterns consistent with `src/lib/queries/assets.ts` and `src/lib/queries/work-orders.ts`

**Create `src/lib/actions/schedules.ts`:**
- Add `"use server"` directive at top
- Import from `zod/v4` (not `zod`) per project convention
- `createSchedule(_prevState, formData)` — requireRole(["manager"]). Validate with Zod: name (string 1-255), description (optional string), assetType (optional, one of assetTypeEnum values), assetId (optional coerce number), frequency (enum: weekly/monthly/quarterly/annual), season (enum: spring/summer/fall/winter/year_round), isSafetyCritical (boolean from "on"/"off" checkbox value), nextDueAt (required date string). Insert into maintenanceSchedules with createdById from session. Call logAudit with action="create", entityType="schedule". revalidatePath("/schedules"). Return { success: true, id }.
- `updateSchedule(id, _prevState, formData)` — requireRole(["manager"]). Same validation. Update the schedule. logAudit action="update". revalidatePath("/schedules").
- `deleteSchedule(id)` — requireRole(["manager"]). Soft delete (set deletedAt=new Date(), isActive=false). logAudit action="delete". revalidatePath("/schedules").
- `generateDueWorkOrders()` — requireRole(["manager", "crew", "inspector"]) (any authenticated user triggers it). Find all active schedules where nextDueAt <= now and deletedAt is null. For each:
  1. Check if open WO (status in created/assigned/in_progress) already exists for this scheduleId — skip if yes
  2. Check seasonal applicability using SEASON_MONTHS — if schedule.season !== "year_round" and current month is not in SEASON_MONTHS[schedule.season], advance nextDueAt but don't create WO
  3. Create work order: title=`${schedule.name} -- ${format(dueDate, "MMM d, yyyy")}` (use date-fns format), type="preventive", priority=schedule.isSafetyCritical ? "high" : "normal", status="created", assetId=schedule.assetId, dockId from asset relation, scheduleId=schedule.id, dueDate=schedule.nextDueAt, createdById=session.userId
  4. Advance nextDueAt using getNextDueDate() helper that adds interval to CURRENT nextDueAt (not now, not completedAt): weekly->addWeeks(1), monthly->addMonths(1), quarterly->addMonths(3), annual->addYears(1)
  5. If nextDueAt was far in the past, loop advancing until nextDueAt > now (catching up missed periods without creating WOs for each)
  6. logAudit action="auto_generate", entityType="work_order" for each generated WO
  7. revalidatePath("/schedules") and revalidatePath("/work-orders")
