**Create `src/lib/queries/compliance.ts`:**
- `getComplianceStats(period: "month" | "quarter" | "year")` — Calculate periodStart and periodEnd based on period. Query workOrders where type="preventive" and scheduleId is not null and dueDate within period:
  - totalDue: count of all such WOs
  - completedOnTime: count where status in (completed, verified) AND completedAt <= dueDate
  - overdue: count where dueDate < now AND status NOT in (completed, verified)
  - compliancePercent: totalDue === 0 ? null : Math.round((completedOnTime / totalDue) * 100)
  - Return `{ totalDue, completedOnTime, overdue, compliancePercent }` where null means "N/A"
- `getComplianceSchedules(period: "month" | "quarter" | "year")` — Return all active schedules with: name, frequency, lastCompletedAt, nextDueAt, isSafetyCritical, assetName (from join), dockName (from join), and per-schedule compliancePercent (same formula scoped to that schedule's WOs in the period). Include status field: "overdue" if nextDueAt < now, "due_soon" if nextDueAt within 7 days, "on_track" otherwise.
- `getAuditTrail(filters: { entityType?: string; userId?: number; dateFrom?: string; dateTo?: string; page?: number })` — Paginated query on auditLogs, 25 per page. Join users for user name. Filter by entityType, userId, dateFrom, dateTo. Order by createdAt desc. Return `{ results, total, page, pageSize }`.
- `getComplianceReportData(period)` — Aggregate data for PDF: compliance stats, overdue items list (schedule name, days overdue, asset), completion rates by category (using ASSET_TYPE_TO_CATEGORY), safety-critical items status, recent audit trail entries (last 50 in period).
- `getAssetHistoryData(assetId)` — All work orders for asset (with parts, time), all audit logs for entity_type="asset" or entityId matching asset's work orders, all schedules linked to asset.
