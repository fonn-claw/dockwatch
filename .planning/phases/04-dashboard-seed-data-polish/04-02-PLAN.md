Then create `src/lib/queries/dashboard.ts` with three new query functions:

**1. `getDashboardHealthScores()`**
Query all active, non-deleted maintenance schedules with their asset and dock relations. For each schedule, determine if it's on-track (nextDueAt > now) or overdue/due-soon (nextDueAt <= now). Apply criticality weights based on asset type:

```typescript
const CRITICALITY_WEIGHTS: Record<string, number> = {
  electrical_pedestal: 3,
  fire_extinguisher: 3,
  fuel_pump: 3,
  piling: 2,
  cleat: 2,
  bumper: 2,
  gangway: 2,
  water_connection: 2,
  dock_light: 2,
  other: 1,
};
```

Score formula: `sum(weight * isOnTrack) / sum(weight) * 100` where isOnTrack = 1 if nextDueAt > now, else 0.

Return `{ marinaWide: number, byDock: Array<{ dockId: number, dockName: string, dockCode: string, score: number }> }`.

**2. `getUpcomingMaintenance()`**
Query active schedules with nextDueAt within a 60-day window (30 days past for overdue, 30 days future). Return array of `{ date: Date, status: "overdue" | "due_soon" | "on_track", scheduleName: string }`. Status: overdue if nextDueAt < now, due_soon if within 7 days, on_track otherwise.

**3. `getRecentActivity(limit: number = 10)`**
Query audit_logs ordered by createdAt desc, limited to `limit`. Join with users for userName. Return array of `{ id: number, action: string, entityType: string, entityId: number | null, userName: string, createdAt: Date, metadata: Record<string, unknown> | null }`.

All functions use the `db` import from `@/lib/db` and schema from `@/lib/db/schema`.
